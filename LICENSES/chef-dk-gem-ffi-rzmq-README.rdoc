<h2><a id="user-content-future-support" class="anchor" href="#future-support" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>FUTURE SUPPORT</h2>

<p>Most people writing Zeromq applications in Ruby should be using the rbczmq
project. It wraps the CZMQ binding which is a much higher-level library for
writing Zeromq code. Find the Ruby gem here:</p>

<pre>http://github.com/methodmissing/rbczmq</pre>

<p>Few projects need to write the low-level zeromq code that this gem allows.
With the release of ffi-rzmq 2.0.3, this library is going into permanent 
maintenance mode. As new versions of libzmq are released, interested
parties should send pull requests to this project or its related project
ffi-rzmq-core to support new features.</p>

<p>The original README text follows…</p>

<h2><a id="user-content-description" class="anchor" href="#description" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>DESCRIPTION:</h2>

<p>This gem wraps the ZeroMQ networking library using the ruby FFI (foreign
function interface). It's a pure ruby wrapper so this gem can be loaded
and run by any ruby runtime that supports FFI. That's all of them: MRI
1.9.x, Rubinius and JRuby.</p>

<p>The Ruby API provided by this gem is <b>NOT</b> very Ruby-like. It very
closely tracks the libzmq C API. However, the contributors to this project
have done all of the hard work to wrap up all of libzmq and make it
accessible from Ruby. If you want it to be more Ruby-like (e.g. raise
Exceptions instead of returning integer codes) then *wrap this library*
with your own and release it as a gem. We will all be grateful!</p>

<p>This single gem supports 0mq 3.2.x and 4.x APIs. The 0mq project started
making backward-incompatible changes to the API with the 3.1.x release. The
gem auto-configures itself to expose the API conforming to the loaded C
library. 0mq 2.x is no longer supported. 0mq API 3.0 is <b>not</b>
supported;  the 0mq community voted to abandon it.</p>

<p>The impetus behind this library was to provide support for ZeroMQ in JRuby
which has native threads. Unlike MRI, which has a GIL, JRuby and Rubinius
allow for threaded access to Ruby code from outside extensions. ZeroMQ is
heavily threaded, so until the MRI runtime removes its GIL, JRuby and
Rubinius will likely be the best environments to run this library.</p>

<p>Please read the History.txt file for a description of all changes,
including API changes, since the last release!</p>

<h2><a id="user-content-performance" class="anchor" href="#performance" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>PERFORMANCE:</h2>

<p>Check out the latest performance results:</p>

<pre>http://www.zeromq.org/bindings:ruby-ffi</pre>

<p>The short version is that the FFI bindings are a few microseconds slower
than using a C extension.</p>

<h2><a id="user-content-featuresproblems" class="anchor" href="#featuresproblems" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>FEATURES/PROBLEMS:</h2>

<p>This gem needs more tests. This gem has been battle tested by myself and
others for years, so I am fairly confident that it is solid. However, it is
inevitable that there will be bugs, so please open issues for them here or
fork this project, fix them, and send me a pull request.</p>

<p>The 'ffi' gem has dropped support for MRI 1.8.x. Since this project
relies on that gem to load and run this code, then this project also no
longer supports MRI 1.8.x. I recommend JRuby for the best performance and
stability.</p>

<p>All features are implemented.</p>

<p>0mq API v3.2-4 client code:</p>

<pre>require 'rubygems'
require 'ffi-rzmq'

if ARGV.length &lt; 3
  puts "usage: ruby local_lat.rb &lt;connect-to&gt; &lt;message-size&gt; &lt;roundtrip-count&gt;"
  exit
end

bind_to = ARGV[0]
message_size = ARGV[1].to_i
roundtrip_count = ARGV[2].to_i

ctx = ZMQ::Context.new
s   = ctx.socket ZMQ::REP
rc  = s.setsockopt(ZMQ::SNDHWM, 100)
rc  = s.setsockopt(ZMQ::RCVHWM, 100)
rc  = s.bind(bind_to)

roundtrip_count.times do
  msg = ""
  rc  = s.recv_string msg
  raise "Message size doesn't match, expected [#{message_size}] but received [#{msg.size}]" if message_size != msg.size
  rc  = s.send_string msg, 0
end</pre>

<p>0mq API v3.2-4 server code:</p>

<pre>require 'rubygems'
require 'ffi-rzmq'

if ARGV.length &lt; 3
  puts "usage: ruby remote_lat.rb &lt;connect-to&gt; &lt;message-size&gt; &lt;roundtrip-count&gt;"
  exit
end

def assert(rc)
  raise "Last API call failed at #{caller(1)}" unless rc &gt;= 0
end

connect_to = ARGV[0]
message_size = ARGV[1].to_i
roundtrip_count = ARGV[2].to_i

ctx = ZMQ::Context.new
s   = ctx.socket ZMQ::REQ
rc  = s.connect(connect_to)

msg = "#{ '3' * message_size }"

time_start = Time.now

roundtrip_count.times do
  assert(s.send_string(msg, 0))

  msg = ''
  assert(s.recv_string(msg, 0))

  raise "Message size doesn't match, expected [#{message_size}] but received [#{msg.size}]" if message_size != msg.size
end

time_end = Time.now
puts "Time #{( time_end - time_start )}"</pre>

<h2><a id="user-content-requirements" class="anchor" href="#requirements" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>REQUIREMENTS:</h2>

<pre>* 0mq 3.2.x, 4.x or later; 2.x, 3.0.x and 3.1.x are no longer supported</pre>

<p>The ZeroMQ library must be installed on your system in a well-known
location like /usr/local/lib. This is the default for new ZeroMQ installs.</p>

<p>If you have installed ZeroMQ using brew, you need to `brew link zeromq`
before installing this gem.</p>

<pre>* ffi (&gt;= 1.0.0)</pre>

<p>* ffi-rzmq-core</p>

<p>This is a requirement for MRI and Rubinius. JRuby has FFI support built in
as a standard component. Do <b>not</b> run this gem under MRI with an old
'ffi' gem. It will crash randomly and you will be sad.</p>

<h2><a id="user-content-install" class="anchor" href="#install" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>INSTALL:</h2>

<p>Make sure the ZeroMQ library is already installed on your system. We
recommend 'brew' or 'macports' to get it.</p>

<pre>% gem install ffi-rzmq # should grab the latest release</pre>

<p>To build from git master:</p>

<pre>% git clone git://github.com/chuckremes/ffi-rzmq
% cd ffi-rzmq
% gem build ffi-rzmq.gemspec
% gem install ffi-rzmq-*.gem</pre>

<p>NOTE for Windows users! In order for this gem to find the libzmq.dll, it
<b>must</b> be on the Windows PATH. Google for “modify windows path” for
instructions on how to do that if you are unfamiliar with that activity.
That DLL also requires that you copy libstdc++-6.dll and
libgcc_s_sjlj-1.dll from DevKit MinGW into the same folder that you copied
libzmq.dll.</p>
